SQL Meurtre mystérieux.

Peux-tu trouver le coupable ?

Il y a eu un meurtre à SQL City ! Le mystère du meurtre SQL est conçu pour être à la fois une leçon autodirigée pour apprendre les concepts et les commandes SQL et un jeu amusant pour les utilisateurs SQL expérimentés pour résoudre un crime intrigant.

Vous débutez en SQL ?

Cet exercice se veut plus un moyen de mettre en pratique les compétences SQL qu'un didacticiel complet. Si vous n'avez jamais utilisé du tout SQL try the walkthrough. Si vous voulez vraiment en savoir beaucoup sur SQL, vous préférerez peut-être un tutoriel complet comme Select Star SQL.

Si vous êtes à l'aise avec SQL, vous pouvez plonger ci-dessous!

Les détectives SQL expérimentés commencent ici

Un crime a eu lieu et le détective a besoin de votre aide. Le détective vous a donné le rapport sur la scène du crime, mais vous l'avez perdu. Vous vous souvenez vaguement que le crime était un assassiner qui a eu lieu quelque temps sur Jan.15 2018 et qu'il a eu lieu dans SQL ville . Commencez par récupérer le rapport de scène de crime correspondant dans la base de données du service de police.

Explorer la structure de la base de données

Les utilisateurs SQL expérimentés peuvent souvent utiliser des requêtes de base de données pour déduire la structure d'une base de données. Mais chaque système de base de données a différentes manières de gérer ces informations. Le mystère du meurtre SQL est construit à l'aide de SQLite. Utilisez cette commande SQL pour rechercher les tables dans la base de données Murder Mystery.

En plus de connaître les noms de table, vous devez savoir comment chaque table est structurée. La façon dont cela fonctionne dépend également de la technologie de base de données que vous utilisez. Voici comment procéder avec SQLite.
